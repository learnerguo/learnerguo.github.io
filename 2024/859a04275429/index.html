1. 类的整个生命周期

flowchart LR
    A["加载"]
    subgraph B["链接"]
        direction LR
        B1["验证"]
        B2["准备"]
        B3["解析"]
        B1-->B2-->B3
    end
    C["初始化"]
    D["使用"]
    E["卸载"]
    A-->B-->C-->D-->E

接下来看看每个阶段做的事情

1.1. 加载阶段完成的事情

  • 根据类的全限定名读取类的二进制流
  • 将字节流所代表的的静态存储结构转化为方法区的运行时数据结构 字节码结构是存储在方法区
  • 在堆区中生成一个代表这个类的java.lang.Class对象，用于方法区这个类的各种数据的访问入口， Class对象是在堆中

1.2. 链接阶段

1.2.1. 准备阶段完成的事情

  • 给类变量分配内存并附初始值(零值)
  • final 修饰的变量直接赋值，而不是初始化为零值

1.2.2. 解析阶段完成的事情

  • 虚拟机将常量池内的符号引用替换为直接引用

1.3. 初始化阶段完成的事情

  • 为类变量赋值（在准备阶段是给类变量分配内存和赋零值），这里是设置为代码中设定的值，比如 static int a = 10, 这里的赋值就是将10赋值给 a
  • 当一个类中含有类变量或静态方法，jvm 会给当前类生成一个 类初始化方法 <clinit>

2. 类加载器

在前面讲类的整个生命周期时，第一个阶段就是加载，加载动作就是通过类加载器完成的，类加载器完成的事情就是 通过类的全限定名获取描述类的二进制流

2.1. jdk9 之前的类加载器

flowchart TB
    A["BootStrap ClassLoader"]
    B["xtension ClassLoader"]
    C["Application ClassLoader"]
    D1["自定义类加载器 User ClassLoader"]
    D2["自定义类加载器 User ClassLoader"]
    A-->B-->C
    C-->D1
    C-->D2

2.2. jdk9 之后的类加载器

flowchart TB
    A["BootStrap ClassLoader"]
    B["Platform  ClassLoader"]
    C["Application ClassLoader"]
    D1["自定义类加载器 User ClassLoader"]
    D2["自定义类加载器 User ClassLoader"]
    A-->B-->C
    C-->D1
    C-->D2

2.3. 双亲委派模式

  1. 当一个类加载器收到加载请求时会先把加载请求委托给父类加载器
  2. 如果父类加载器也存在父类加载器，继续向上委托，直至最顶层引导类加载器
  3. 如果父类加载器能完成类的加载，就交给父类加载器完成，反之由子类进行加载

注：这里类加载器中所说的父类不是继承关系，而是实现类加载器的层级关系，子类加载器中会包含父类加载器的引用而已

2.3.1. 优点

  • 避免核心类库被修改，即使有相同全限定类名的类，也会先交给父类去判断
  • 避免全限定名相同的类被重复加载，导致程序异常
