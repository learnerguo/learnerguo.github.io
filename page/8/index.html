发表于2023-11-28|存储Redis
缓存雪崩概念缓存雪崩是指在缓存服务器重启或者大量 key 集中在某一时刻同时失效，而这些缓存在同一时刻被大量的请求所访问，从而引起大量的缓存集中失效，引起请求的性能急剧下降。 解决方案 避免缓存设置相近的有效期 对数据进行预热 缓存击穿概念缓存击穿是指一个热点 key在缓存失效的时候，同时大量的请求都在访问这个key，而这些请求都在同时失效，从而引起大量的请求集中到达数据库，从而引起数据库的压力。 解决方案 热点数据永远不过期，即在数据过期之间可以先进行更新 使用分布式锁，只让一个线程去查询 db 并返回结果，不过这种方式在高并发下比较影响性能 缓存穿透概念缓存和数据库中都没有的数据，还是有源源不断的请求到达，导致每次请求都会到数据库，从而压垮数据库 解决方案 对于查询数据库也不存在的数据，依然可以存放在 redis 中，并且设置一个较短的过期时间 使用布隆过滤器
