Mysql配置项和配置文件

以下基于 Mysql8 版本

Mysql 程序选项

Mysql 是一个 客户端-服务端 程序，安装 Mysql 后 bin 目录下有很多可执行程序，比如

  • 启动服务端的 Mysqld 程序
  • 启动服务端的 mysqld_safe 脚本程序，这个脚本也会启动 Mysqld, 但会包含更多功能
  • 启动客户端的 Mysql 程序

还有很多可执行程序，具体可以参考 Mysql programs

这些可执行程序在启动时都可以指定参数，这些参数有两种形式

  • 长格式
    • 以 -- 开头
    • 如果对应的参数有值，则是 --key=value 形式，比如 --port=3306， 注意 对应的 key, = 和 value 之间不要有空格， 比如 --port = 3306 形式是错误的
  • 短格式
    • 以 - 开头
    • 如果对应的参数有值，则是 -key value 形式，比如 -p 3306， 参数和值之间不需要 =

配置文件

通过命令行执行可执行文件并且在命令后面增加参数是一次性有效的，如果停止了 Mysql 服务，再次启动需要重新指定参数，所以可以将参数写在配置文件，这样就可以不用每次都指定参数了。

Mysql配置文件官方文档说明

Unix 和 类 Unix 下配置文件读取位置

File NamePurpose
/etc/my.cnfGlobal options
/etc/mysql/my.cnfGlobal options
SYSCONFDIR/my.cnfGlobal options
$MYSQL_HOME/my.cnfServer-specific options (server only)
defaults-extra-fileThe file specified with –defaults-extra-file, if any
~/.my.cnUser-specific options
~/.mylogin.cnfUser-specific login path options (clients only)
DATADIR/mysqld-auto.cnfSystem variables persisted with SET PERSIST or SET PERSIST_ONLY (server only)
  • $MYSQL_HOME 是环境变量，可以自己设置
    1. Ubuntu中设置环境变量的几种方式
  • 以 ~ 开头的则是用户目录下的配置文件
  • defaults-extra-file 是指通过命令行指定的配置文件，比如 mysqld --defaults-extra-file=/usr/loca/mysql.cnf

配置文件包含的内容

下面是一个 my.cnf 配置文件的示例

主要包含以下几部分内容

  • 注释和空行
  • [mysql]， [mysqld] 这种在官方文档中称为 group, 也就是配置文件被分组了，组里面的内容就是可执行文件名
    • 不同组下的配置项是给不同的启动命令使用的，如果选项组名称与程序名称相同，则组中的选项将应用于该程序， 比如 [mysql] 组下面的选项会在使用 mysql 命令时读取
    • [server] 组下配置项将作用于所有的服务器程序
    • [client] 组下配置项将作用于所有的客户端程序
    • [mysqld_safe] 和 [mysql.server] 这两个程序在启动时除了读取自己组下的配置还会读取 [mysqld] 下的配置
  • 组下面的具体配置
    • 配置文件中的配置项只能是长格式，而且不需要 -- 开头

配置文件优先级

不同配置文件的优先级

不同目录下都有配置文件，并且有相同的配置项的情况下，以后读取的为准，上面已经列出了读取的目录

File NamePurpose
/etc/my.cnfGlobal options
/etc/mysql/my.cnfGlobal options
SYSCONFDIR/my.cnfGlobal options
$MYSQL_HOME/my.cnfServer-specific options (server only)
defaults-extra-fileThe file specified with –defaults-extra-file, if any
~/.my.cnUser-specific options
~/.mylogin.cnfUser-specific login path options (clients only)
DATADIR/mysqld-auto.cnfSystem variables persisted with SET PERSIST or SET PERSIST_ONLY (server only)

如果 /etc/my.cnf 和 ~/.my.cn 下都有相同的配置，则使用的是 ~/.my.cn 配置文件中的内容

相同配置文件的优先级

  1. mysql 配置文件有多个 group
  2. 若是在不同的 group 中出现相同的配置则以最后一个 group 中出现的配置为准（前提是对应的程序会读取这个 group 的配置）
