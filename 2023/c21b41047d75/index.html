redis 分布式锁要做到的事情

  1. 独占性： 任何时刻只能有一个线程持有
  2. 高可用： 若在 redis 集群环境下，不能因为某一个节点挂了而出现获取锁和释放锁失败的情况
  3. 防止死锁： 要有超时控制机制或者撤销操作
  4. 正确性： 不能 unlock 别人的锁，自己的锁只能自己释放
  5. 可重入
